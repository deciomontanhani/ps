### O que é MVP?

MVP é a versão mais enxuta possível de um produto focada em testar e validar hipóteses com *early adopters*, consumidores mais tolerantes às potenciais deficiências de um MVP.

Tipos:
- *Concièrge*: Focado em receber 1º clientes; não é possível escalar, mas conhecer a fundo seus clientes.
- Mágico de Oz: Zero código até se entender como o cliente se comporta; equipe faz tudo manualmente por trás do sistema.
- Vídeo explicativo: um vídeo.
- *Crowdfunding*: lance sua ideia num site de financiamento coletivo; caso se arrecade o valor mínimo, desenvolva!
- Fumaça: um simples anúncio ou *landing page* do produto/serviço.

### Estratégias de precificação

Tipos:
- Premium: apps pagos no *download*.
  - :+1: Menor stress com taxas de conversão
  - :+1: Fácil de mensurar esforços de marketing
  - :-1: Difícil competição com apps grátis
  - :-1: Alta expectativa dos usuários
- Ads: propagandas dentro do app.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Fácil implementação
  - :-1: Pode compremeter a experiência do usuário
- Freemium: download grátis, mas *upgrade* pago.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Possível cobrar mensalmente
  - :+1: Usuário pode testar antes de comprar, sem comprometê-lo
  - :-1: Difícil definir uma experiência boa para o app grátis e o pago
  - :-1: Alta expectativa dos usuários pagos, tirando a foco do app grátis
- Ecommerce: venda de produtos físicos dentro do app.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Margem de lucro maior que lojas físicas
  - :+1: Mais fácil de iniciar que lojas físicas
  - :+1: Funciona bem para venda de nichos
  - :-1: Logística
  - :-1: Concorrência com grandes *marketplaces*
- In-app purchasing: venda digital dentro do app.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Margem de lucro alta
  - :+1: Sem necessidade de logística
  - :+1: Pode ser combinado com outros modelos
  - :-1: Lojas de apps ficam com parte da receita
  - :+1: Poucos realmente compram (2~3%)
- Híbrido: ads + in-app purchasing.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Experiência específica para cada tipo de usuário
  - :-1: Segmentação dos tipos de usuários pode ser desafiador
- Subscription: similar ao freemium, mas se cobra pelo conteúdo.
  - :+1: Permite disponibilizar o app de graça
  - :+1: Mais fácil que o premium para criar uma grande base de usuários
  - :+1: Possível cobrar mensalmente
  - :+1: Usuário pode testar antes de comprar, sem comprometê-lo
  - :-1: Difícil definir uma experiência boa para o app grátis e o pago
  - :-1: Alta expectativa dos usuários pagos, tirando a foco do app grátis

### Técnica de pirata (AARRR)

- Aquisição: de que canais os usuários vêm?
- Ativação: quantos usuários tem uma boa primeira experiência?
- Retenção: eles voltam e revisitam seu serviço/produto?
- Recomendação: eles indicam a outros?
- Receita: você consegue monetizar a partir desse comportamento?

### Growth hacking

Mover mais rápido uma maior quantidade de usuários de um estágio do AARRR para o próximo.

### Canvas

![Modelo do Canvas](http://www.consultorio20.com.br/wp-content/uploads/2016/02/canvas-business-model.png)

- Clientes
  - Segmentos de cliente: perfil dos clientes.
  - Canais de vendas: como se alcança os clientes.
  - Relacionamento com o cliente: serviços de comunicação para manter o cliente.
- Produto/serviço
  - Oferta de valor: o seu serviço e o que ele vale para o cliente.
  - Atividades chave: o que você precisa ser feito para o seu modelo funcionar.
  - Recursos chave: o que você precisa ter para o seu modelo funcionar.
- Parceiros
  - Parceiros chave: os parceiros.
- Dinheiro
  - Fonte de receitas: como e com o que você ganha dinheiro
  - Fonte de custos: como e com o que você gasta dinheiro

### SWOT

- Força: interno e bom.
- Fraqueza: interno e ruim.
- Oportunidades: externo e bom.
- Ameaças: externo e ruim.
